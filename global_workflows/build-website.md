# Webサイト構築ワークフロー
## ～ 見栄えのするサイトを最速で公開する6ステップ ～

---

## Steps

### 1. デザイン要件ヒアリング（3分）

ユーザーに以下を一括で確認する（1問ずつ聞かない）：

```
以下をまとめて教えてください：
1. サイトの目的（例：商品紹介・問い合わせ獲得・自己PR）
2. ターゲット層（例：20〜30代の会社員）
3. 好みのデザイン感（例：シンプル・モダン・温かみ）
4. 必要なセクション（例：ヒーロー・特徴・料金・問い合わせ）
5. カラーのイメージ（なければ「おまかせ」でOK）
```

### 2. サイト構成・カラーパレット提案

// turbo
回答をもとに以下を自動生成する：

```markdown
## 構成案（承認してから実装に進みます）

### セクション構成
- ヒーロー：キャッチコピー + CTA
- 〇〇：[ユーザーの要望から自動生成]
- フッター

### カラーパレット
- メイン：#[色コード]
- アクセント：#[色コード]  
- テキスト：#[色コード]
- 背景：#[色コード]

### スタイル方針
- [ヒアリング内容をもとに3行で説明]
```

「この構成で進めますか？」と確認する

### 3. HTML骨格の実装

// turbo
承認を受けたら骨格を自動実装：

```
作成するファイル：
- index.html（メインページ）
- style.css（スタイル）
- script.js（インタラクション）
- images/（画像フォルダ）
```

実装後：「骨格を作りました。ブラウザで確認してください」と伝える

### 4. コンテンツ・デザイン実装

以下の順序で実装する：
1. ヒーローセクション（一番重要・最初に仕上げる）
2. 各セクション（構成順に）
3. フッター
4. ホバーアニメーション・スムーズスクロールなどUX改善

コピーライティングは以下のパターンで書く：
```
見出し：「[悩み]」で困っていませんか？
サブ：[解決策を1行で]
説明：[具体的なメリット3点]
CTA：[動詞 + 具体的な内容]（例：「無料で試してみる」）
```

### 5. モバイル対応・最適化

// turbo
以下を自動チェック・修正する：

```
□ メディアクエリの設定（768px・1024px）
□ 画像のmax-width: 100% 設定
□ フォントサイズのclamp()による流動的設定
□ タップターゲットのサイズ（最低44px）
□ 読み込み速度の確認
```

### 6. 公開準備・デプロイ

以下を自動生成・設定する：

```html
<!-- 自動設定するメタタグ -->
<meta name="description" content="[サイトの説明]">
<meta property="og:title" content="[タイトル]">
<meta property="og:description" content="[説明]">
<meta property="og:image" content="[OGP画像]">
```

デプロイ方法を説明する：
```
【Vercelを使う場合（おすすめ）】
1. vercel.com でアカウント作成（無料）
2. 「New Project」→ フォルダをドラッグ&ドロップ
3. Deploy ボタンを押すだけ → URLが発行される

【GitHub Pagesを使う場合】
1. GitHubにリポジトリを作る
2. Settings → Pages → Source: main branch
3. 数分でhttps://username.github.io/repo-name/ で公開
```

完了報告：
```

- アクセス解析（Google Analytics）の設置
```
