# アプリ開発ワークフロー
## ～ アイデアを動くアプリに変える5ステップ ～

---

## Steps

### 1. ヒアリング（2分）
- ユーザーに「何を・誰が・どこで使うか」を1回で聞く
- 答えをもとに技術スタックを選定する
- 迷ったらシンプルな方を選ぶ（HTML > React > Next.js の順）

### 2. タスクリスト生成（自動）

// turbo
```
task.md を以下の形式で自動生成する：

# アプリ名：[ユーザーが答えた内容から命名]
## MVP機能（最初に作るもの）
- [ ] [機能1]
- [ ] [機能2]
## 追加機能（後で追加するもの）
- [ ] [機能A]
- [ ] [機能B]
## 技術スタック
- [選定した技術を記載]
```

### 3. MVP実装（コアだけ先に動かす）

以下の順序で実装する：

```
実装順序：
1. プロジェクトの骨格（フォルダ構成・基本ファイル）
2. メインの画面/UI
3. コアとなるロジック（一番重要な機能）
4. データの保存・読み込み（必要な場合）
```

実装後、必ずユーザーに確認する：
「実装しました。動作を確認してもらえますか？」

### 4. フィードバック反映・機能追加

// turbo
ユーザーのフィードバックを受けて：
- バグがあれば原因を説明して修正
- 追加機能はタスクリストに追加して順番に実装
- 一度に複数変更しない（1変更ずつ確認）

### 5. 完成・デプロイ準備

以下を自動でやる：

```
□ README.md の生成（使い方・起動方法を日本語で）
□ 不要ファイルの削除
□ .env.example の作成（環境変数がある場合）
□ デプロイ手順の説明（Vercel/Streamlit Cloud など）
```

完了後にユーザーへ報告：
```
✅ アプリが完成しました！

【作ったもの】
- [機能一覧]

【起動方法】
- [起動コマンド]

【次のステップ（任意）】
- [追加できる機能の提案]
```

---

## トラブル時の対応

エラーが発生した場合：
1. エラーメッセージを日本語で解説する
2. 原因を1〜2行で説明する
3. 修正方法を提示してから実行する（いきなり修正しない）
